SCHEMA VALIDATION & SANITIZATION

LLM output is never trusted raw
— every field is validated and normalized

sanitizeDrillOutput() — 7-step normalization (runs before QA)
1
Remove forbidden keys

Strip diagramV1 and singular progression — LLM sometimes uses legacy key names

2
Merge diagram aliases

If only diagramV1 exists, copy to diagram. If both present, diagram wins.

3
Coerce area dimensions to numbers

"30 yards" → 30. "40" → 40. Critical: strings fail QA clarity = 2.

4
Fix age mentions

Regex-replace all age variants to exactly match input.ageGroup — "U-14", "u14", "14-year-olds" → "U14"

5
Normalize progressions to array

If progressions is a string, wrap in array. Merge stray progression entries into array.

6
Convert string organization

If LLM returns organization: "Set up..." (string), parse into object with setupSteps[] extracted.

7
Youth guards + game model enforcement

Apply applyYouthGuards(). Inject model-specific constraints if array is sparse. Enforce ≥8 model cues across session.

Critical Fail Conditions
area = string

lengthYards or widthYards as string → clarity score = 2 → NEEDS_REGEN

players = []

Empty diagram.players array → QA fail → diagram re-enrichment triggered

age mismatch

Any mention of wrong age in drill → clarity ≤ 2 → sanitizer attempts fix first

non-JSON

LLM returns markdown / prose → parseJsonSafe returns null → throw "LLM returned non-JSON"

// parseJsonSafe — strips markdown fences
function parseJsonSafe(text) {
  const clean = text
   .replace(/```json\n?/g, "")
   .replace(/```\n?/g, "").trim();
  const i = clean.indexOf("{");
  const j = clean.lastIndexOf("}");
  if (i === -1 || j === -1) return null;
  try { return JSON.parse(clean.slice(i, j+1)); }
  catch { return null; }
}
// Number coercion for area dimensions
if (typeof area.lengthYards === "string") {
  const n = parseFloat(area.lengthYards);
  area.lengthYards = isNaN(n) ? 30 : n;
}

Goal-Player Alignment (automatic post-processing)

After generation, the system calculates the centroid of all ATT players. If centroid.x > 50 on HORIZONTAL pitch: right goal → teamAttacks = ATT. Goal facingAngle auto-corrected to face attacking direction. Prevents misaligned "which team attacks which goal" errors.
